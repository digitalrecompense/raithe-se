BUG-001: PATH TRAVERSAL IN BACKUP SYSTEM
================================================================================
Priority: CRITICAL
Location: crates/storage/src/backup.rs:51-72
Issue Type: Path Traversal / Directory Traversal
Verification Status: ✅ CONFIRMED

Description: The copy_dir_recursive function joins destination path with filename
without validating that the resulting path stays within the backup directory. An
attacker controlling file names could use "../" to escape the backup directory.

Code Snippet:
--------------------------------------------------------------------------------
fn copy_dir_recursive(src: &Path, dst: &Path) -> std::io::Result<()> {
    // ...
    for entry in std::fs::read_dir(src)? {
        let entry = entry?;
        let src_path = entry.path();
        let dst_path = dst.join(entry.file_name());  // <-- No validation
        // ...
    }
}
--------------------------------------------------------------------------------

Verified Location: backup.rs lines 51-72
Code confirms: No path validation - dst_path joined with file_name() directly.

Impact: An attacker who can control file names in the source directory could write
files to arbitrary locations on the filesystem. With write access to /data/backups,
an attacker could overwrite /etc/cron.d or ~/.ssh/authorized_keys.

CVSS 3.1 Score: 8.1 (HIGH) - Network exploitable, no privileges required

Recommended Fix: Canonicalize both paths and verify destination stays within
backup directory using Path::canonicalize().

--------------------------------------------------------------------------------

BUG-002: INTEGER OVERFLOW IN DOC ID COUNTER
================================================================================
Priority: CRITICAL
Location: crates/indexer/src/lib.rs:241-243
Issue Type: Integer Overflow
Verification Status: ✅ CONFIRMED

Description: The doc_id counter uses AtomicU64::fetch_add without checking for
overflow. After 2^64 documents, the counter wraps to 0, causing doc_id collisions.

Code Snippet:
--------------------------------------------------------------------------------
let doc_id_u64 = self
    .doc_id_counter
    .fetch_add(1, std::sync::atomic::Ordering::SeqCst);
--------------------------------------------------------------------------------

Verified Location: indexer/src/lib.rs lines 261-263
Code confirms: fetch_add without overflow check.

Impact: Doc_id collisions cause index corruption. When multiple documents have
the same ID, search results become unpredictable and can leak sensitive content
from one document into another document's search results.

CVSS 3.1 Score: 7.5 (HIGH) - Network exploitable, no privileges required

Recommended Fix: Check for overflow and return error instead of wrapping.

Impact: Connection exhaustion - attacker holds connections open until server runs
